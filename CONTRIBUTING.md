# Contributing to Omnilertlab Website
=====================================

Thank you for considering contributing to the Omnilertlab Website. This guide will help you get started with the process.

## Fork Guide
-------------

To contribute to the Omnilertlab Website, follow these steps:

1. **Fork the repository**: Click the "Fork" button in the top-right corner of the repository page to create a copy of the repository in your own GitHub account.
2. **Clone the repository**: Clone the forked repository to your local machine using `git clone https://github.com/your-username/omnilertlab.git`.
3. **Create a new branch**: Create a new branch for your feature or bug fix using `git checkout -b feature/your-branch-name`.
4. **Make changes**: Make the necessary changes to the code and commit them using `git add .` and `git commit -m "your commit message"`.

## Branch Naming
----------------

When creating a new branch, please use the following naming conventions:

* **Feature branches**: `feature/your-branch-name`
* **Bug fix branches**: `fix/your-branch-name`
* **Release branches**: `release/your-branch-name`

## Commit Style
----------------

When committing changes, please follow the conventional commit message format:

* **Type**: Use one of the following types: `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `build`, `ci`, `chore`
* **Description**: Write a brief description of the changes made

Example: `git commit -m "feat: add new feature to homepage"`

## Pull Request Template
-------------------------

When submitting a pull request, please use the following template:

### Pull Request Title
* Use a descriptive title that summarizes the changes made

### Pull Request Body
* Describe the changes made and the reasoning behind them
* Include any relevant screenshots or examples
* List any dependencies or libraries used

## Code of Conduct
------------------

This project adheres to the [Omnilertlab Code of Conduct](https://github.com/omnilertlab/.github/blob/main/CODE_OF_CONDUCT.md). By participating in this project, you agree to abide by its terms.
